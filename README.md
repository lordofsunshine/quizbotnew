# Quiz!

Quiz! - это Discord-бот для викторин, таблицы лидеров и магазина ролей за очки. Бот получает вопросы из OpenTDB, переводит их на русский язык, проводит ручные и автоматические викторины, начисляет очки за правильные ответы и показывает статистику пользователей. 

Этот бот печально известен тем, что его код украл другой разработчик, взломал бота через токен, выгнал всех людей из технической поддержки Quiz! и запустил рассылку о том, что данный бот был продан другому владельцу и ссылка на другой сервер технической поддержки. Проект был украден и аудитория переехала на чужой сервер. Спустя 2 года, решил опубликовать данный проект на GitHub в публичной форме, так как имею на это право, поскольку я был главным разработчиком этого бота. Дальнейшую судьбу "украденного" проекта я не знаю.

Я тогда искал других разработчиков, которые помогут с созданием новых функций в боте, один такой разработчик откликнулся, была небольшая беседа, передал ему код проекта, но забыл удалить конфиг, где хранился в том числе токен бота. Взлом произошёл ночью, логично почему. Но, для меня тогда это было уроком на всю жизнь.

#### Код был слегка переписан, приведён в порядок, всё-таки, проект теперь публичный. Бот протестирован.

## Возможности

- ✅ Slash-команда `/quiz` запускает викторину в текущем канале.
- ✅ Авто-викторины можно настроить через `/auto-quiz`.
- ✅ Поддерживаются категории, сложность и несколько раундов.
- ✅ Очки начисляются только за правильные ответы.
- ✅ `/leaderboard` показывает таблицу лидеров по очкам или соотношению побед.
- ✅ `/rank` создаёт карточку профиля пользователя с очками и статистикой.
- ✅ Пользователь может выбрать фон карточки профиля.
- ✅ `/auction` открывает магазин серверных ролей за очки.
- ✅ Администраторы могут добавлять и удалять товары магазина.
- ✅ `/role` настраивает роль, которую бот упоминает перед авто-викториной.
- ✅ `/delete-me` удаляет данные пользователя из базы.
- ✅ `/help` показывает список команд и подробную справку.
- ✅ `/bot` показывает краткую информацию о боте.

## Установка

```bash
npm install
```

Создайте `.env` в корне проекта:

```env
TOKEN=production_discord_bot_token
DEV_TOKEN=development_discord_bot_token
MONGO_URL=mongodb_connection_url
DEV_MODE=
```

Если `DEV_MODE` задан, бот использует `DEV_TOKEN`. В обычном режиме используется `TOKEN`.

## Запуск

```bash
npm start
```

Для разработки:

```bash
npm run dev
```

Запуск тестов:

```bash
npm test
```

Проверка зависимостей:

```bash
npm audit
```
